# UI 변경 구현 기록

작성일: 2026-06-15

## 요청 사항

- 지출 데이터 숫자를 `1,000원`처럼 천 단위 콤마와 원 단위로 표시
- 기존 카드 리스트 페이지 새 구성
- 할인 항목 및 실적 체크 페이지 새 구성

## 로컬 구현 내용

로컬 검증 파일: `outputs/improved-index.html`

### 1. 통화 입력 표시

- 지출 입력 필드를 `text` 기반 통화 입력으로 전환
- 화면 표시 예시: `200,000원`, `1,000원`
- 내부 계산은 `parseMoney`/`getCurrencyValue`로 숫자만 추출해 기존 계산 흐름 유지
- 저장값 복원, 직접 입력, 자동 반영, 사용자 추가 결제처에도 동일 포맷 적용

### 2. 기존 카드 리스트 탭

새 탭: `💳 기존 카드`

구성:

- 현재 선택된 카드별 월 결제처 합계
- 등록 결제처 수
- 전환 후보 수
- 자동납부/고정 결제처 테이블
- 결제처, 현재 카드, 월 금액, 추천 배분, 상태 표시

### 3. 할인 항목·실적 체크 탭

새 탭: `✅ 할인·실적 체크`

구성:

- 카드별 필요 실적
- 실적 인정 예상액
- 실적 미포함 예상액
- 충족/부족 예상 상태
- 주요 할인 항목 체크리스트
- 상품설명서/자동납부 경로 확인 체크 항목

## 반영 제한

GitHub 커넥터는 `index.html` 전체 교체만 지원하고, 현재 GitHub CLI는 로그인되어 있지 않아 170KB급 단일 HTML을 PR 브랜치에 직접 업로드하지 못했다.

GitHub CLI 로그인 후 다음 파일을 `index.html`로 커밋하면 된다.

```powershell
Copy-Item outputs\improved-index.html index.html -Force
git add index.html
git commit -m "feat: improve spending and card check pages"
git push origin codex/drive-data-improvements
```
